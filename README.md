# AED地図案内アプリ(開発中)

現在地から最適なAED設置施設を検索・案内するWebシステム

## 🚀 概要
「いざというときに迷わずAEDを取りに行くための道具」をコンセプトとした、位置情報特化型のWebアプリです。生コードからの脱却とモダンなWebシステム開発の学習を目的としているため、規模に見合わない部分がありながらも、フレームワーク等の技術を採用しています。

## 🛠 技術スタック（学習用メモも兼ねる）

### Backend
- **Laravel 12**: 裏側のロジックを全て管理するフレームワーク。生のPHPで書くと面倒くさいセキュリティ対策やデータベース操作など諸々の処理を簡単に、かつ安全に行うためのもの。
- **PostgreSQL / PostGIS**: 地理空間データの管理を担う。デフォルトはMySQLだったが、空間位置情報の計算などに適しているPostGISに変更。PostGISはPostgreSQLのエンジンの上で動くため、セットで入っている。

### Frontend
- **React / TypeScript**: コンポーネント指向による堅牢なUI開発
- **Inertia.js**: Laravel と React をつなぐ架け橋。従来はこれらを連携させるためのAPIの開発が必須だったが、Inertiaの導入により、その手間を大幅に削減できる。
- **Vite**: 高速プレビュー担当。コードを書き替えた瞬間、一瞬でブラウザに反映させるためのツール。GoogleMAPの倍率や中央の座標を変えた際、すぐにブラウザの画面でプレビューが反映されていたのはこいつのおかげ。

### Infrastructure / API
- **Docker (Laravel Sail)**: 開発環境のコンテナ化。別端末からのプルリクエスト等の操作を学びたいと考えていたため、別端末にDockerを入れるだけで開発・実行環境が整う状態を作りたかった。
- **Google Maps Platform**: 地図描画・ルート案内
- **@vis.gl/react-google-maps**: React向けGoogle Mapsライブラリ。Googleが公式に推奨しており、かつ新規プロジェクトであったため、従来のgoogle-map-react や react-google-mapsを差し置いて選定した。

## ✨ こだわり・学習のポイント

- **地理空間情報の高度な扱い**
  単なる座標保存ではなく、PostGISを採用することで「半径〇m以内のAEDを抽出」といった計算をデータベース側で効率化しました。まだ距離計算の機能は実装していません。（2026-03-09）
- **型安全な開発**
  TypeScriptを導入し、バックエンドから渡るデータの型定義を行うことで、バグの抑制と開発速度の向上を図っています。
